# 待测试

- Codex App 插件文档：新增插件安装教程，区分 AI 自动安装、手动安装和本仓库开发调试安装；同步优化插件 README 与技能说明。
- 配置与用户偏好：每个模型渠道新增 OpenAI / NewAPI / 百炼 / Gemini 调用格式选择，默认 OpenAI，切换默认格式时同步更新默认 Base URL；渠道 Tab 增加紧凑醒目的提醒，说明模型是否可选需要到“模型”Tab 选择。
- Gemini 调用格式：支持拉取 Gemini 模型列表，并用于文本问答、在线 Agent 工具调用、基础生图和图生图请求。
- 音频和视频生成：选择 Gemini 调用格式时先给出不支持提示，仍需使用 OpenAI、NewAPI 或百炼格式渠道。
- NewAPI 调用格式：与 OpenAI 格式仅在视频接口上不同，创建任务为 `POST /v1/video/generations`、查询任务为 `GET /v1/video/generations/{task_id}`。响应支持字符串 code 信封（`{code:"success",message,data}`）和数字 code 信封，自动拆包；状态与视频地址读取兼容已拆信封/未拆信封两种形态，成功状态识别 `SUCCESS`/`completed`/`succeeded` 等，视频地址从 `result_url`/`url`/`data.output.video_url` 等位置兜底。成功后直接返回视频 URL 由播放器播放（不下载 blob，避免第三方 CDN 无 CORS 头导致卡住）。图生视频统一用 `images` 数组字段（单张图也用 `images:[url]`，否则会被 NewAPI 当成文生视频）。OpenAI 格式仍保留原有 `POST /v1/videos` 三步流程。
- 视频创作台按钮状态：修复 `refreshLogs` 在每次保存日志时都重新恢复“生成中”日志、导致当前任务轮询成功后 `running` 仍为 true、按钮不可点击的问题；改为仅在页面挂载时恢复未完成日志。
- 百炼调用格式（阿里云 DashScope）：默认 Base URL `https://dashscope.aliyuncs.com`，鉴权 `Authorization: Bearer`。生图为同步接口 `POST /api/v1/services/aigc/multimodal-generation/generation`，文生图 content 传 `text`、图生图/编辑 content 加 1-3 个 `image` 项（base64 data URL），响应从 `output.choices[].message.content[].image` 取图并转成 data URL 存本地。生视频为异步 `POST /api/v1/services/aigc/video-generation/video-synthesis`（`X-DashScope-Async: enable`），图生视频用 `input.media` 的 `first_frame` 传图，轮询 `GET /api/v1/tasks/{task_id}`，`SUCCEEDED` 取 `output.video_url`（直接返回 URL 不下载 blob）。百炼暂不支持文本问答、Agent 工具调用、音频生成和拉取模型列表。**已知限制**：百炼 `/api/v1/tasks/` 端点不返回 CORS 头（而生图/视频创建的 `/services/aigc/` 端点返回），浏览器直连视频轮询会被跨域拦截；百炼视频需将渠道 Base URL 指向一个反代 dashscope 并加 CORS 头的代理，生图/图生图不受影响。**开发态**：`web/vite.config.ts` 已配置 `server.proxy`，把 `/bailian` 转发到 `https://dashscope.aliyuncs.com`，开发时把百炼渠道 Base URL 填 `/bailian` 即可跑通视频轮询（仅 `npm run dev` 生效）。**生产态**：项目 `nginx.conf` 已内置 `/bailian/` 反代 location（转发到 dashscope 并加 CORS 头），Docker 部署后把百炼渠道 Base URL 填 `https://站点/bailian` 即可。**百炼视频多图模式**：按模型族发参数（kling: `mode`/`aspect_ratio`/`duration`/`audio`/`watermark`；wan: `resolution`/`duration`/`prompt_extend`/`watermark`）；`input.media` 按参考图模式组合——首帧 `first_frame`、首尾帧 `first_frame+last_frame`、参考生视频 `refer`（仅 `kling/kling-v3-omni-video-generation` 支持多图）；视频设置面板新增"参考图模式"下拉（自动/首帧/首尾帧/参考生视频，自动=按图片数量：1→首帧、2→首尾帧、3+→参考生视频）和"输出"开关组（生成声音/添加水印，生成声音仅 kling 系列生效）。
